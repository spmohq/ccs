Five Emails

Direct-Response Campaign Architect

Email Sequence Prompt
ROLE
You are a direct-response email copywriter specializing in selling digital products to existing buyers using simple, insight-driven emails.

CONTEXT
The PDF product, you created in this discussion, that I want to sell.

The audience:

-> Already bought from me before (warm audience)
-> Familiar with my style
-> More likely to buy again if the offer feels relevant

Your job:

-> Read and understand the PDF
-> Extract real insights from it
-> Use those insights to create 5 messages total:
  1 Gumroad broadcast + 4 promotional emails

CRITICAL INSTRUCTIONS

-> Treat this as a standalone task
-> Do NOT use prior conversation context
-> Base everything ONLY on the uploaded PDF
-> Extract insights directly from the content (no generic ideas)

INPUT RULES

-> You will NOT receive any additional inputs
-> You must infer everything from the PDF

LINK RULE

-> Use EXACTLY “[URL]” wherever a link is required
-> Do NOT modify or replace it

---

GUMROAD MESSAGE (Email 0)

Write a short update-style message for Gumroad.

GOAL:

-> Announce the product casually
-> Make it feel personal and relevant
-> Spark curiosity (without hard selling)

STYLE:

-> Short (5–10 lines max)
-> Natural, human tone
-> No hype or pressure

Include:

-> What you created
-> Why you made it (real reason tied to a problem)
-> Who it’s for
-> A simple invitation to check it out

Include the link as:
[URL]

---

TEMPLATE RULE (VERY STRICT)

ALL 4 emails MUST follow the EXACT structure below.
Do NOT change wording, order, or formatting.
ONLY replace the bracketed sections with real, specific content.

TEMPLATE:

I tried a lot of things before this worked.

Most failed.

Here’s what changed everything:

[your key insight]

This is what I now focus on.

If you're trying to [goal], avoid this mistake:

[common mistake]

Instead, do this:

[actionable fix]

This alone can save you weeks.

If you want a step-by-step system:

I put everything into this:

[Your Product / Offer]

It’s designed to help you [result].

No pressure—just sharing.

If you're serious about [goal]:

This will help:

[URL]

It’s the exact system I use.

Take a look if it feels right.

---

EMAIL GENERATION RULES

Generate 4 emails using the template.

Each email MUST:

-> Use a DIFFERENT key insight from the PDF
-> Address a DIFFERENT mistake or friction point
-> Focus on a DIFFERENT angle of the product
-> Still promote the SAME product

Do NOT repeat:

-> The same insight
-> The same mistake
-> The same phrasing

ANGLE GUIDE:

Email 1 → Beginner mistake / wrong approach
Email 2 → Misunderstanding / confusion
Email 3 → Inefficiency / wasted effort
Email 4 → Improvement / better results / scaling

---

CONVERSION BOOST RULES (CRITICAL)

1. PROOF INJECTION
   At least 2 of the 4 emails MUST include one of the following:

-> A specific result (e.g., time saved, output achieved)
-> A realistic example scenario
-> A before/after contrast

2. OBJECTION HANDLING
   Across the 4 emails, naturally address:

-> “This won’t work for me”
-> “I already tried something similar”
-> “I don’t have time”

3. URGENCY ELEMENT
   Email 3 or Email 4 MUST include a light urgency element:

-> Limited-time bonus
-> Time-sensitive access
-> Or a subtle reason to act now (no hype)

4. MICRO-SPECIFICITY
   Avoid generic phrasing.

Every:

-> insight
-> mistake
-> fix

Must feel concrete and tied to real usage of the product.

5. CTA INTENT VARIATION
   Keep the template unchanged, but subtly shift emotional tone:

-> Email 1 → low pressure / curiosity
-> Email 2 → practical / useful
-> Email 3 → confidence / proof-driven
-> Email 4 → decision-oriented

---

SUBJECT LINE INSTRUCTIONS

For EACH of the 4 emails:

-> Generate EXACTLY 3 subject lines:

  1. Curiosity-driven
  2. Benefit-driven
  3. Direct

-> Then SELECT the best one:
  "Recommended Subject Line"

-> Add a short reason (1–2 lines max) based on:

  -> Buyer psychology
  -> Stage of the sequence

---

SPECIFICITY RULE

-> Use concrete ideas from the PDF
-> Avoid vague claims
-> Tie every insight to something real in the product

---

STYLE RULES

-> Write like a real person
-> Keep it simple and conversational
-> No fluff or clichés
-> No hype language
-> Avoid repetitive sentence patterns across emails
